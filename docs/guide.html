vanilla.waves Guide

You give it a number. It gives you a number back.
Same idea as p5.waves — only here you point that number at an element, not a pixel. No p5, no canvas.

Install

One script tag. No npm, no build step, zero dependencies.

<script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/vanilla.waves@v0.1.0/vanilla.waves.min.js"></script>

That one file is the whole library: the wave math (a byte-identical port of p5.waves — 1009/1009 parity checks against v3.4.0) plus a tiny DOM engine (one shared 30 fps loop, offscreen pause, reduced-motion support). It exposes a global VanillaWaves (alias vWaves, and Waves if nothing else claimed it).

Your First Element

The declarative path: drop an element with a data-wv type. The engine scans the page on load and wires it up — no code of your own.

<div class="wv-eq" data-wv="eq" data-count="24"></div>

You'll see a row of bars whose heights ride a wave that morphs every few seconds. data-wv names the element type; any other data-* attribute is passed to that type as an option.

The built-in demo types (eq, sliders, progress, checks, glyphs, ring, load, walk, trio) live in docs/examples.js — the prototype catalog for the upcoming vanilla.waves_elements CDN. The core library ships the math + engine; you register the element types (or load that catalog).

Two Ways to Use It

Declarative for decoration, imperative for control. Both call the exact same math.

1 — Imperative: drive any property yourself

Make a sampler once, then read a number out of it inside your own loop. This is the heart of the library — the rest is sugar.

const s = VanillaWaves.createSampler({ shift: true, range: [-50, 50] });

function frame(t) {
  // t is whatever number you pass — seconds, scroll, anything
  box.style.transform = 'translateY(' + s.sample(0, t) + 'px)';
  requestAnimationFrame(frame);
}
requestAnimationFrame(frame);

2 — Declarative: register a type, let the engine drive it

Tell the engine how to build and update an element type; it handles the loop, the timing, the offscreen pause.

VanillaWaves.register('nudge', {
  create(node, opts, h) {
    return { sampler: h.makeSampler({ range: [-50, 50] }) };
  },
  update(state, t) {
    node.style.transform = 'translateY(' + state.sampler.sample(0, t) + 'px)';
  }
});
// then: <div data-wv="nudge"></div>  — auto-wired on load

What Can a Number Drive?

Anything in the DOM that takes a number. A few ideas:

// position
el.style.transform = 'translateX(' + s.sample(0, t) + 'px)';

// opacity
el.style.opacity = VanillaWaves.wave(i, { wave: 'pulse', range: [0.2, 1] });

// a real form control's value
range.value = VanillaWaves.wave(i, { wave: 'sine', range: [0, 100] });

// colour channel
const g = VanillaWaves.wave(x, { wave: 'meta sine', range: [0, 255] });
swatch.style.background = `rgb(${g},${g},${g})`;

// text content
cell.textContent = ramp[Math.floor(VanillaWaves.wave(i, { range: [0, ramp.length - 1] }))];

If a property accepts a number, a wave can drive it.

Wave Shift — The Fun Part

One flag, and your wave starts switching to random formulas on its own. Smooth morphs. Different sequence every reload.

const s = VanillaWaves.createSampler({
  shift:         true,     // that's the flag
  range:         [0, 100],
  shiftInterval: 4,        // hold each wave 4 units of t
  shiftDuration: 1.5       // morph over 1.5 units of t
});

// inside your loop — read the live state for free:
s.sample(0, t);   // the number
s.waveName;       // "mountain peaks"
s.shifting;       // true while morphing
s.targetName;     // "batman" — what's coming next
s.mix;            // 0 -> 1 during a morph

Shift has no internal clock — it reads the t you pass. Tie t to performance.now()/1000 for real seconds, or to scroll position, and shift follows that instead.

Closing rings while shifting — group: 'closing'

Lay dots around a ring with transform: rotate() translateY() and a shifting wave normally tears its seam open the moment it lands on a new formula — every wave has its own period. The 'closing' pool fixes that: all its waves share one base period (6.2832), so a sweep of sampler.period × lobes closes seamlessly through every morph.

const ring = VanillaWaves.createSampler({ shift: true, group: 'closing', amplitude: 1 });
const LOBES = 6, N = 56;

function frame(t) {
  const sweep = ring.period * LOBES;   // read straight off the sampler
  for (let i = 0; i < N; i++) {
    const frac = i / N;
    const r = 62 + ring.sample(frac * sweep, t) * 26;
    dots[i].style.transform = `rotate(${frac * 360}deg) translateY(-${r}px)`;
  }
  requestAnimationFrame(frame);
}

Experimental Period values may drift by ~0.001 in minor versions. Perfect for visuals; not for plotters or anything that minds sub-unit drift.

The Engine

The declarative layer. It runs one shared requestAnimationFrame loop for every data-wv element on the page, capped at 30 fps (a DOM node doesn't need 60). It pauses elements that scroll offscreen (IntersectionObserver) and renders a single static frame when the user prefers reduced motion.

API

CallWhat it does
register(name, def)Define an element type. def = { create(node, opts, h), update(state, t) }.
init(target?)Scan and wire elements. No arg = all [data-wv]. Also accepts an element, a selector, or a NodeList.
destroy(target)Stop and clean up. The shared loop stops itself when the last element is gone.

Types must be registered before the engine auto-inits on DOMContentLoaded. Register later or add elements dynamically? Just call VanillaWaves.init() again — already-wired elements are skipped.

Helpers passed to create(node, opts, h)

HelperWhat it does
h.makeSampler(extra)A sampler with element-friendly defaults (shift on, gentle, low frequency), merged with the element's data-* options and your extra.
h.num(v, d)Parse a data-* string to a number, with a default.
h.norm(v)Clamp a ~[-1,1] value to [0,1].
h.el(tag, cls)Create an element with an optional class.

Every element gets aria-hidden="true" (it's decorative) and a wv--ready class once wired. Each picks its own start time from its seed, so a wall of identical elements doesn't move in lockstep.

Build Your Own Element Type

This is the whole point — and the seed for vanilla.waves_elements. A type is two functions: create builds the DOM and returns state; update writes wave values into it each frame.

VanillaWaves.register('dots', {
  create(node, opts, h) {
    const n = h.num(opts.count, 20), dots = [];
    for (let i = 0; i < n; i++) dots.push(node.appendChild(h.el('span')));
    return { dots, n, sampler: h.makeSampler({ range: [0, 1] }) };
  },
  update(state, t) {
    for (let i = 0; i < state.n; i++) {
      state.dots[i].style.opacity = state.sampler.sample(i * 0.4, t);
    }
  }
});

Now <div data-wv="dots" data-count="40"></div> works anywhere on the page. That's a publishable element.

Copy-Paste Starters

Just a number

const y = VanillaWaves.wave(x, 'mountain peaks');

A moving element

const s = VanillaWaves.createSampler({ shift: true, range: [-40, 40] });
function frame(t) { el.style.transform = 'translateY(' + s.sample(0, t) + 'px)'; requestAnimationFrame(frame); }
requestAnimationFrame(frame);

A wave-driven slider

const s = VanillaWaves.createSampler({ shift: true, range: [0, 100] });
range.value = s.sample(0, t);   // a real <input type="range">

Binary field on two samplers

const rowS = VanillaWaves.createSampler({ shift: true, range: [-1, 1], seed: 1 });
const colS = VanillaWaves.createSampler({ shift: true, range: [-1, 1], seed: 2 });
box.checked = (rowS.sample(row * 5 + t, t) + colS.sample(col * 2.5 - t, t)) > 0;
