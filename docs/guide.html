guide

Everything you need to use vanilla.waves - the wave dialect with no p5 and no <canvas>. The math mirrors p5.waves v3.4.0 exactly; on top of it sits a tiny DOM engine that drives plain HTML.

Install

vanilla.waves is a single file with no runtime dependencies. No npm, no bundler, no build step.

Route A: CDN (jsDelivr)

One tag pulls the math (waves-core.js) plus the DOM engine (engine.js):

<script src="https://cdn.jsdelivr.net/gh/seb-prjcts-be/vanilla.waves@v0.1.0/vanilla.waves.min.js"></script>

Route B: Local

Copy vanilla.waves.min.js (or vanilla.waves.js for the readable build) next to your HTML and point a <script> at it. Want only the math, no engine? Load waves-core.js on its own.

global The library exposes VanillaWaves (alias vWaves, and Waves as a drop-in when p5.waves isn't present).

Your first element

Register an element type once, mark elements with data-wv="<type>". The shared loop calls your update every frame; offscreen elements pause automatically.

<div data-wv="bars" data-seed="13" data-count="20"></div>

<script src="vanilla.waves.min.js"></script>
<script>
  VanillaWaves.register('bars', {
    create(node, opts, h) {
      const n = h.num(opts.count, 16);
      const bars = [];
      for (let i = 0; i < n; i++) bars.push(node.appendChild(h.el('span')));
      return { bars, s: h.makeSampler() };        // state
    },
    update(state, t) {                             // every frame
      state.bars.forEach((b, i) => {
        const v = 0.1 + state.s.sample(i * 0.5, t) * 0.9;
        b.style.transform = 'scaleY(' + v.toFixed(3) + ')';
      });
    }
  });
  // auto-inits on DOMContentLoaded; call VanillaWaves.init() after adding nodes
</script>

Types must be registered before the engine's auto-init (DOMContentLoaded). If you register or add nodes later, call VanillaWaves.init() again - it skips elements already wired.

The math: three ways to call it

1. Lazy: wave(y)

One argument. A deterministic value from seed 0 at amplitude 100. Same input, same output.

const v = VanillaWaves.wave(x);   // amplitude 100, wave picked by seed 0

2. Quick: by name or seed

const a = VanillaWaves.wave(x, 'mountain peaks');  // by name
const b = VanillaWaves.wave(x, 42);                // by seed (deterministic pick)

3. Full control: options object

const y = VanillaWaves.wave(x, {
  wave: 'bumpy sine',
  amplitude: 80,
  frequency: 1,
  t: performance.now() / 1000,
  phase: 0,
  seed: 7
});

Options

Every option, its default, and what it does. All are optional.

Start here

OptionDefaultDescription
wavenullA name, an index (0…33), or [a, b] to morph. null → picked by seed.
amplitude100Output is normalised to [−1, 1] and scaled to [−amp, +amp]. Default wave(y) swings ±100.
frequency1Multiplier on x. Higher = faster cycles.
t0Time, in seconds. Anything that moves reads this.
seed0Picks the wave when wave is null; seeds the PRNG for harsh waves.

Level up

OptionDefaultDescription

mix0.5Morph position (0…1) when two waves are set.

createSampler(): cache for tight loops

Calling wave(x, opts) hundreds of times per frame re-resolves the wave, group, and stats every call. createSampler() resolves all of that once and hands you a fast sample().

const s = VanillaWaves.createSampler({
  shift: true,
  group: 'gentle',
  amplitude: 120
});

// per frame:
const v = s.sample(x, t);   // position + time

Use sample(y, t) when shift or time-based motion is on; sample(y) when t doesn't matter (a morph sampler also takes sample(y, t, mix)). A shift sampler exposes live getters: waveIndex, waveName, targetName, shifting, mix, period, targetPeriod. A non-shift sampler is static and carries only waveIndex, waveName and period.

Engine API

The DOM engine adds three calls to the same global. Core stays pure - the engine only wires DOM to it.

CallDoes
register(name, def)Define an element type. def.create(node, opts, h) returns state; def.update(state, t) mutates the DOM each frame.
init(target?)Wire up [data-wv] - no arg for all, or a selector / element / NodeList.
destroy(target?)Stop and clean up (e.g. when loading finishes). Give the type an optional def.destroy(state, node) hook to also remove the DOM that create built.

Helpers passed to create: makeSampler(extra?) (a sampler with element-friendly defaults - shift on, gentle, low frequency), norm(v) (maps ≈[−1,1] to [0,1]), num(v, default), el(tag, class?).

Conventions: marker data-wv, ready class wv--ready, shared loop at 30 fps, all data-* become opts, data-speed scales time. CSS class and element names use a wv- prefix - never dots (p5.waves.loader is not a valid selector).

Wave shift

The killer feature: set shift: true and the library picks a wave, holds it for shiftInterval seconds, then morphs into the next over shiftDuration seconds. Forever.

const s = VanillaWaves.createSampler({
  shift: true,
  shiftInterval: 2,    // hold each wave 2s
  shiftDuration: 1.5,  // morph over 1.5s
  group: 'gentle',
  amplitude: 80
});

s.waveName;    // "mountain peaks"
s.targetName;  // "batman" (what's coming next)
s.shifting;    // true while morphing
s.mix;         // 0 → 1 during the morph

note Shift is not deterministic across page loads: a per-session random offset is mixed in, so the same seed yields a different sequence each reload (identical within one session). This is faithful to p5.waves and intentional - the per-load variation is richer than a fixed one.

Closing shapes

When you sweep a wave around a ring, the last point rarely lands on the first - you get an ugly seam. The fix is to sweep over an integer multiple of the wave's period: the x-distance over which wave(x) repeats. Every periodic wave has a measured period, listed on the Waves page.

Shift and closed shapes normally fight: a ring tears its seam open the moment it lands on a new wave, because every formula has its own period. The 'closing' pool ends that - its 17 waves all share one base period (62.8319 = 2π/0.1), so a sweep of sampler.period × lobes closes seamlessly through every transition.

// Read the sweep straight off the sampler.
const ring = VanillaWaves.createSampler({
  shift: true, group: 'closing', amplitude: 30
});
const sweep = ring.period * 8;   // 8 lobes, holds across every shift

sampler.period returns the current wave's measured period (the shared base period for a closing pool, null for non-periodic waves). Full canvasless SVG-ring recipe on the Waves page.

experimental Period values may drift by ~0.001 in minor versions. Perfect for visuals; not for plotters or anything that gets angry about sub-unit drift.

Morph between two waves

Pass two names, control the blend with mix from 0 to 1.

const y = VanillaWaves.wave(x, {
  wave: ['classic sine', 'mountain peaks'],
  mix: 0.5   // 0 = first wave, 1 = second
});

Sweep mix over time for a smooth crossfade between two specific shapes - useful when shift is too wild for the look you want.

Wild mode

One dial, unpredictability (0…1), adds noise modulation on top of any wave. At 0 you get the clean wave; at 1, chaos that still respects the wave's envelope.

const y = VanillaWaves.wave(x, {
  wave: 'mountain peaks',
  mode: 'wild',
  unpredictability: 0.7
});

Parity & drift

waves-core.js is the canonical p5.waves.js math with only the 8-line p5 prototype hook removed. It tracks p5.waves as the source of truth - bit-identical output, verified across 1009 parity checks (all 34 waves, morph, range, wild, sampler). A weekly drift-watch compares the two so the port can never silently fall behind. Dialect baseline: p5.waves v3.4.0.

Time is just a number: you pass t in seconds - the library never calls a clock itself. So you can scrub time, run faster than real-time, run backwards, or freeze a moment.

Copy-paste starters (canvasless)

Animated bars

<div id="bars" style="display:flex;align-items:flex-end;gap:3px;height:100px"></div>
<script>
  const box = document.getElementById('bars');
  const bars = Array.from({length: 24}, () => {
    const b = document.createElement('span');
    b.style.cssText = 'width:8px;height:100%;background:#14140f;transform-origin:bottom';
    return box.appendChild(b);
  });
  const s = VanillaWaves.createSampler({ shift: true, range: [0, 1] });
  (function tick(now){
    const t = now / 1000;
    bars.forEach((b, i) => {
      const v = 0.1 + s.sample(i*0.5, t) * 0.9;   // range [0,1] keeps scaleY positive
      b.style.transform = 'scaleY(' + v.toFixed(3) + ')';
    });
    requestAnimationFrame(tick);
  })(0);
</script>

Wave as opacity (breathing element)

const s = VanillaWaves.createSampler({ wave: 'classic sine', range: [0.15, 1] });
(function tick(now){
  el.style.opacity = s.sample(0, now / 1000);
  requestAnimationFrame(tick);
})(0);

Wave as hue (colour drift)

const s = VanillaWaves.createSampler({ wave: 'bumpy sine', range: [0, 360] });
(function tick(now){
  el.style.backgroundColor = 'hsl(' + s.sample(0, now / 1000) + ' 70% 55%)';
  requestAnimationFrame(tick);
})(0);

Shift-driven text loader

const ramp = ' ·:-=+*#%@';
const s = VanillaWaves.createSampler({ shift: true, range: [-1, 1] });
(function tick(now){
  const v = s.sample(0, now / 1000);
  const i = Math.max(0, Math.min(ramp.length-1, Math.floor((v+1)/2 * ramp.length)));
  el.textContent = ramp[i];
  requestAnimationFrame(tick);
})(0);

Morph between two waves

(function tick(now){
  const mix = (Math.sin(now * 0.0005) + 1) / 2;
  const v = VanillaWaves.wave(0, {
    wave: ['classic sine', 'triangle'], mix, range: [0, 100]
  });
  bar.style.height = v + '%';
  requestAnimationFrame(tick);
})(0);
